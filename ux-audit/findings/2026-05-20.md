# Drippin UX 진단 리포트 — 2026-05-20

> Playwright MCP 브라우저로 직접 탐방. **코드는 일절 수정하지 않음.** 산출물은 `ux-audit/` 폴더(스크린샷·리포트)뿐이며, 폼은 흐름·레이아웃만 점검하고 제출하지 않음.

## 탐방 목표

> 처음 온 사용자가 5초 안에 서비스(드립커피 레시피·일지 공유)의 가치를 이해하고, 2클릭 이내로 핵심 기능(레시피 보기 / 일지 작성)에 자연스럽게 도달한다.

## 환경
- 뷰포트: 기본 390px(모바일), 768px(태블릿) 교차 점검
- dev 서버: `http://localhost:3000` (200)
- 로그인: Playwright 프로필이 새 세션이라 로그인이 풀려 있어, 탐방 중 사용자가 직접 카카오 로그인(김민성 / Drippin) → 로그인 필요 페이지까지 진단 완료

## 페이지별 진단 (성공 기준 체크리스트)

### 1. 홈 `/` (피드)
- [ ] **첫 화면에서 서비스가 무엇인지 5초 안에 이해** — ❌ 미달. 상단 sticky 바에 **레시피/일지 탭만** 있고 로고·서비스명·한 줄 가치 제안이 전혀 없음. 처음 온 사용자는 커피 카드 나열만 보고 "여기가 뭐 하는 곳인지"를 추론해야 함. (`components/home/HomeWrapper.tsx`의 상단 바는 `role="tablist"`만 렌더)
- [x] 핵심 기능까지 2클릭 이내 — 레시피 카드 클릭(1) → 상세, 상세에서 "레시피 진행하기"(2). 일지 작성은 하단탭 일지(1) → "새로운 일지 작성하기"(2). 양호.
- [x] 모바일 390px 레이아웃 깨짐 없음, 하단 탭 88px 패딩 적용(`pb-[88px]`).
- [x] 768px에서도 max-w 컨테이너 중앙 정렬 + 양옆 여백, 깨짐 없음.
- [△] 콘솔: Amplitude API 키 누락 에러(dev), LCP 이미지 priority 경고. 기능 영향 없음.
- 비고: 진입 즉시 **만족도 조사 모달**이 자동 노출됨(`before/home-with-survey-modal.png`). 첫 방문/가치 인지 전에 뜨면 이탈 요인이 될 수 있음.

### 2. 레시피 상세 `/recipe/37`
- [x] CTA 명확 — 히어로 하단 "레시피 진행하기" 단일 1차 액션. 좋음.
- [x] 좋아요 수·시간·ICE/HOT·작성자·설명 정보 구조 깔끔.
- [△] 하단 "이 레시피를 사용한 일지" 섹션이 **비어 있을 때 빈 상태 카피 없이 헤딩만** 노출.

### 3·4. 레시피 온보딩 → 타이머
- [x] 파라미터(드리퍼/필터/비율/온도/분쇄도/메모) 한 화면에 정리 → "시작하기".
- [x] 타이머: 진행 단계 점(dot) 인디케이터 + "50초 동안 40g 붓고 기다려주세요" 지시 + 원형 타이머 + 재생/다시하기. **완성도 높음.**
- [△] 온보딩 분쇄도 슬라이더가 '곱게'에 위치하나 메모는 '800~1000마이크론'(=보통~약간 굵게) → 표시 불일치 가능성(콘텐츠/데이터).

### 5·6. 레시피 추가(4단계) / 일지 추가(2단계)
- [x] 레시피 추가 상단 4단계 스텝 인디케이터(기본정보/물과원두/추출/추가정보) 명확. 일지 추가는 내용+사진첨부 → 다음.
- [ ] **비로그인 사용자 안내** — ❌. 비로그인 상태로 진입 시 LoginNudge가 아니라 **폼이 잠깐 그대로 렌더된 뒤** `useEffect`에서 `router.back()`/`router.push("/")`로 튕김(보호 콘텐츠 flash + 갑작스러운 이동). (`app/recipe/add/page.tsx:337-346`, `app/log/add/page.tsx:173-180`)
- [ ] **콘솔 에러** — ❌. 레시피 추가 진입 시 **react-select 하이드레이션 mismatch** 에러 발생: `Prop did not match. Server: "react-select-2-live-region" Client: "react-select-3-live-region"`. (`components/share/CreatableSelector.tsx:33`, react-select에 고정 `instanceId` 미지정)

### 7. 내정보 `/my`
- [x] 비로그인: "서비스를 이용하려면 로그인이 필요해요" + 카카오/구글 버튼. LoginNudge 자체는 깔끔(`before/my.png`). → **이 패턴을 add 페이지에도 재사용하면 좋음.**
- [x] 로그인: 이름/이메일/닉네임(변경하기)/로그아웃 정상.

### 8·9·10. 목록 & 일지 상세
- [x] 레시피 목록·일지 목록 모두 "새로운 ○○ 작성하기" 플로팅 버튼 제공.
- [△] **정보구조 불일치**: 홈 일지 탭 피드는 사진을 크게 노출하나, `/log` 목록은 **썸네일 없이 텍스트만**.
- [△] `/log` 목록이 **항목 전체(89개+)를 한 번에 렌더**(목록에 페이지네이션/무한스크롤 미관측). 데이터 증가 시 스크롤 길이·렌더 비용 증가.
- [△] 목록에 **검색·필터·정렬 부재** — 데이터 증가 시 탐색성 저하.
- [x] 일지 상세: 사진 + 본문 + 해시태그 + 날짜 + (소유자) 케밥 메뉴. 양호.

## 우선순위별 개선 제안

### 🔴 High
1. **홈 첫 화면에 서비스 가치 제안 추가** — 문제: 5초 가치 이해 불가(탭만 노출). 제안: 상단에 로고/서비스명 + 한 줄 태그라인(예: "드립커피 레시피와 시음 일지를 기록·공유하는 공간") 또는 비로그인 한정 온보딩 히어로(가치 1줄 + "레시피 둘러보기"/"일지 시작하기" CTA). 관련: `components/home/HomeWrapper.tsx`, `app/page.tsx`, (신규) 헤더 컴포넌트.
2. **add 페이지의 비로그인 처리 개선** — 문제: 보호 폼이 flash 후 강제 이동(이질감, 로그인 유도 실패). 제안: `/my`의 LoginNudge 컴포넌트를 재사용해 비로그인 시 폼 대신 로그인 안내를 노출(또는 진입 가드로 전환). 관련: `app/recipe/add/page.tsx:337-346`, `app/log/add/page.tsx:173-180`.

### 🟡 Medium
3. **CreatableSelector 하이드레이션 mismatch 제거** — 문제: 레시피 추가 진입 시 콘솔 에러 다수. 제안: `react-select`에 고정 `instanceId`(예: `instanceId="dripper"`) 지정해 SSR/CSR id 일치. 관련: `components/share/CreatableSelector.tsx:33`.
4. **`/log` 목록 정보구조 정리** — 문제: 홈 피드(사진)와 목록(텍스트) 불일치 + 전체 일괄 렌더. 제안: 목록에 썸네일 추가로 일관성 확보, 페이지네이션/무한스크롤 적용. 관련: `app/log/page.tsx`, `components/share/LogCard.tsx`.
5. **LCP 이미지 priority 부여** — 문제: 첫 화면 이미지 priority 누락 경고 → 모바일 첫 로드 체감 저하. 제안: 피드 상단 첫 카드 이미지에 `priority`. 관련: `components/share/RecipeCard.tsx:117`.

### 🟢 Low
6. 레시피 상세 "이 레시피를 사용한 일지" 빈 상태 카피 추가. 관련: `app/recipe/[id]/`.
7. 만족도 조사 모달 노출 타이밍 재검토(첫 방문 즉시 대신 일정 행동/방문 후). 
8. 목록 검색·필터·정렬 도입(데이터 증가 대비).
9. 온보딩 분쇄도 슬라이더 위치 ↔ 메모(마이크론) 일치성 점검.
10. (운영 무관) dev 환경 Amplitude API 키 설정으로 콘솔 노이즈 제거.
