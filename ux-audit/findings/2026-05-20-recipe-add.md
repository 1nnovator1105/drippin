# 레시피 추가 위저드 UX 진단 — 2026-05-20

> Playwright로 4단계 위저드(`/recipe/add`)를 로그인 후 직접 끝까지 입력하며 진단. **데이터는 제출하지 않음**(게시하기 직전까지만).

## 플로우 요약

| 단계 | 내용 | 비고 |
|------|------|------|
| 1. 기본 정보 | 레시피 이름* / 드리퍼 / 필터 / 온도(아이스·핫)* | CreatableSelector로 직접 입력 가능 |
| 2. 물과 원두 | 커피·물 양 + 온도 / 분쇄도 슬라이더 / 메모 | 커피·물 입력 시 **1:9 비율 실시간 표시** |
| 3. 추출 | 블루밍 네/아니오 / 붓는 횟수 / 단계별 입력 다이얼로그 | 타임라인 + 푸어별 모달(물 g·시간·옵션·메모) |
| 4. 추가 정보 | 레시피 설명* / 사진 | `게시하기`는 설명 입력 전 비활성 |

스크린샷: `ux-audit/screenshots/before/add-step1-filled.png` ~ `add-step4-extra.png`

## 잘 된 점
- 4단계 진행 인디케이터 항상 노출 → 현재 위치/남은 단계 명확
- 2단계 **실시간 비율(1:9) 계산** — 우수한 피드백
- 3단계 **단계별 입력 다이얼로그**로 복잡한 푸어 데이터 분할 — 영리한 설계
- 필수값 게이팅(2·3·4단계 `다음`/`게시하기` 비활성)으로 불완전 제출 방지
- 친절한 보조 카피("목록에 없는 드리퍼는 직접 입력할 수 있어요")

## 개선 포인트 (우선순위)

### 🔴 High
1. **증분 입력 ↔ 누적 표시 혼동 (3단계)**
   - 다이얼로그는 "물을 몇 그램 부었나요?"(=이번에 부은 양, 예: 140g)를 묻는데, 타임라인은 **누적(180g/2:00)**으로 표시 → "140 넣었는데 왜 180?" 혼란 가능
   - 제안: 타임라인에 "누적" 라벨 명시 또는 증분/누적 병기
   - 파일: `components/recipe/InfoTimeline.tsx`
   - 근거: `add-step3-complete.png` (블루밍 40g/0:30 → 1차 추출 180g/2:00)
2. **숫자 입력 기본값이 `0` (placeholder 아님)**
   - 2단계 양/온도, 3단계 다이얼로그 모두 실제 값 `0`이 채워져 있어, 탭 후 타이핑 시 `040`처럼 됨(전체 선택/삭제 필요)
   - 제안: 기본값을 빈값 + `placeholder="0"`
   - 파일: `app/recipe/add/page.tsx`, `app/recipe/[recipeId]/edit/EditRecipe.tsx`, 푸어 다이얼로그(`InfoTimeline.tsx`)

### 🟡 Medium
3. **"추출 완료"가 빨간색 (3단계)** — 검증 에러처럼 보이나 실제론 종료 노드. 중립/브랜드색으로. (`InfoTimeline.tsx`)
4. **물 온도가 '비율' 그룹에 묶임 (2단계)** — "커피와 물의 **비율**" 헤딩 아래 온도까지 포함. 온도는 비율과 무관 → 별도 라벨 분리 또는 헤딩 조정. (`app/recipe/add/page.tsx`)

### 🟢 Low
5. **3단계 학습 부담** — "다음 푸어까지 몇 초" 등 초보자에 생소. 프리셋("기본 4:6 레시피 불러오기")·도움말로 진입장벽↓ (로드맵 "저장된 레시피 불러오기"와 연결).
6. **1단계 필수값 검증 확인 필요** — 1단계 `다음`이 항상 활성처럼 보임. 이름/온도 미입력 시에도 진행되는지 점검.
