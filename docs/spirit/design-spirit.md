# 디자인 정신 (Design Spirit)

> **변하지 않는 것.** 구체적 토큰·컴포넌트는 변하지만([`../engineering/architecture.md`](../engineering/architecture.md#디자인-토큰)), 아래 감각은 화면을 만들 때마다 기준으로 남는다.

---

## 1. 모바일이 먼저다 (max 768px)

기본 캔버스는 모바일이다. 컨테이너는 **max 768px**, 그 이상에서는 콘텐츠를 중앙 정렬하고 양옆 여백을 둔다.
데스크톱은 "모바일을 넓은 화면에서 본 것"이지, 별도 레이아웃이 아니다.

## 2. 정보 밀도의 균형 — 인스타 like

글이 **너무 많아 답답하지도, 너무 적어 빈약하지도** 않게. 인스타그램처럼 한눈에 들어오는 밀도를 지향한다.
"정보를 더 넣자"와 "다 빼고 비우자" 사이에서, 항상 **읽기 편한 중간**을 택한다.

## 3. 화면당 1차 액션은 하나

각 화면에는 사용자가 할 **가장 중요한 행동 하나**가 명확해야 한다.
예: 레시피 상세 = `레시피 진행하기`, 목록 = 작성 FAB. 2차 액션은 시각적으로 양보한다.

## 4. 즉각 반응한다

탭/버튼을 누르면 **즉시** 반응이 보여야 한다 — 활성 상태 변화, 로딩 표시(커피 애니메이션), 토스트.
"눌렀는데 아무 일도 안 일어나는 듯한" 순간을 만들지 않는다.

## 5. 따뜻하고 깔끔하게

브랜드는 **따뜻한 커피 브라운**. 과한 장식·강한 그림자보다 **여백과 절제된 라운딩**으로 정돈된 인상을 준다.
색은 디자인 토큰으로만 쓰고, 강조는 브랜드 컬러 하나로 모은다.
