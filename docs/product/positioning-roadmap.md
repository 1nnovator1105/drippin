## 포지셔닝 (토스증권 모델)

**코어는 "내 드립커피 기록·재생 도구", 그 위에 커뮤니티를 얹는다.** 코어(개인 도구)에 더 비중을 두고, 커뮤니티는 무거운 소셜(팔로우/댓글) 없이 **검색·태그·대상별 묶음**으로 가볍게 형성한다.

| 토스증권 | → Drippin |
|---|---|
| 내 주식(매매) | **내 레시피·일지**(기록·재생) ← 코어 |
| 종목별 커뮤니티 | **대상별 묶음**(원두·카페·지역·태그별 일지) |
| 종목 무관 피드 | **일반 커피 피드**(홈 피드) |

---

## A. 코어 — 개인 기록·재생 도구

- 레시피: 피드 조회 ✅ / 생성(4단계, 기존 불러오기) ✅ / 편집 ✅ / 삭제 ✅ / **재생(타이머)** ✅
- 일지: 작성·사진첨부 ✅ / 수정 ✅ / 삭제 ✅
- 마이페이지: 닉네임 변경 ✅
- **내 기록 통계/요약 ✅** — 이번 달 잔 수·누적·자주 마신 곳·자주 쓴 태그·최근 6개월 추이(`/my`의 `MyStats`)
- 레시피↔일지 양방향 연결 강화 🔴 ← *다음 후보*

## B. 커뮤니티 레이어 (가벼운 것부터)

- 검색·필터 ✅ / 좋아요 ✅
- **프로필 홈 ✅** (`/profile/[handle]` → 그 사람의 레시피·일지 모음, handle 고유) — 코어↔커뮤니티 다리
- **태그 페이지 ✅** (`/tag/[tag]` → 해당 태그가 담긴 일지 모음) — "종목별 커뮤니티"의 첫 조각
- 대상별 페이지(원두·카페·지역) 🔴 ← *다음 후보*
- 댓글 🔴 · 팔로우/팔로잉 🔴 · 추천 콘텐츠 🔴 (그 다음 단계)

## ⏸ 보류 (현재 우선순위 아님)

- **프로필 이미지** — 현재 쓰일 화면이 마땅치 않음
- **레시피 프리셋 불러오기** — 제공할 프리셋 콘텐츠 미정
- 본격 소셜(댓글/팔로우/추천) — 콘텐츠·사용자 규모가 더 쌓인 뒤

## 다음 후보 (우선순위)

1. **레시피↔일지 연결 강화** — 일지에서 사용한 레시피로, 레시피에서 그걸로 내린 일지로 양방향 이동.
2. **대상별 페이지 확장** — 태그 페이지에 이어 원두·카페·지역 단위 묶음.
